## 3. 核心设计

### 3.1 统一响应

```java
public class Result<T> {
    private int code;           // 状态码
    private String message;     // 提示信息
    private T data;             // 响应数据
    private long timestamp;     // 响应时间戳
    private String traceId;     // 链路追踪ID

    // 成功响应
    public static <T> Result<T> ok(T data);
    public static <T> Result<T> ok();

    // 失败响应
    public static <T> Result<T> fail(int code, String message);
    public static <T> Result<T> fail(ErrorCode errorCode);
}
```

### 3.2 错误码体系

| 范围 | 类别 | 说明 |
|------|------|------|
| 200 | 成功 | 操作成功 |
| 400-499 | 客户端错误 | 参数错误、未认证等 |
| 500-599 | 服务端错误 | 系统异常 |
| 1000-1999 | 系统模块 | 用户/角色/菜单等 |
| 2000-2999 | 安全模块 | 认证/授权/License |
| 3000-9999 | 业务模块 | 各业务模块错误 |

### 3.3 接口规范

```
URL格式: /{version}/{module}/{resource}[/{action}]

示例:
GET    /v1/system/users           # 用户列表
GET    /v1/system/users/{id}      # 用户详情
POST   /v1/system/users           # 创建用户
PUT    /v1/system/users/{id}      # 更新用户
DELETE /v1/system/users/{id}      # 删除用户
POST   /v1/system/users/export    # 导出用户
```

## 7. 技术决策记录

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 架构模式 | 模块化单体 | 降低复杂度、便于后续拆分 |
| ORM框架 | MyBatis-Plus | 灵活、生态好、学习成本低 |
| 分库分表 | ShardingSphere-JDBC | 透明分片、社区活跃 |
| 数据库迁移 | Flyway | 成熟稳定、Spring Boot集成好 |
| API文档 | SpringDoc | 支持OpenAPI 3.1、配置简单 |
| 权限模型 | RBAC+ABAC混合 | 兼顾管理简单和灵活性 |
