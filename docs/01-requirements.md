# 当前交付范围说明

## 1. 目标

当前版本目标不是一次性交付完整企业平台，而是交付一个可以稳定运行、可以继续扩展、具备基本上线条件的后台管理后端与管理端前端。

## 2. 当前已经交付的能力

- 系统管理：用户、角色、菜单、部门
- 认证中心：登录、退出、用户信息、刷新令牌
- OpenAPI：应用账号、签名校验、防重放、接口授权
- 运维基础：健康检查、监控、告警
- 工程基础：Flyway、统一响应、统一异常、Maven 版本治理

## 3. 当前明确不在首版范围

- License 完整闭环
- ABAC 动态授权完整实现
- 前端动态路由
- 完整 API 门户目录与在线调试
- 多租户、分库分表、分布式事务

## 4. 非功能要求

### 4.1 安全

- 敏感配置不允许直接写入仓库
- OpenAPI 必须校验签名、时间窗、`nonce` 和接口授权范围
- API Secret 不允许通过列表或详情接口常驻返回

### 4.2 可交付性

- 后端必须通过 `mvn -q clean test`
- 前端必须通过 `npm run test` 与 `npm run build`
- 空库启动必须可验证
- 文档必须与当前代码一致

## 5. 验收清单

- 登录成功且可刷新令牌
- 权限不足返回统一 403
- OpenAPI 重放请求被拒绝
- 空库启动成功
- 构建、测试、打包链路通过
