## 1. 项目概述

### 1.1 背景与目标

BML-Backend 是一套**企业级管理系统后端服务**，采用**领域驱动设计(DDD)**与**模块化单体架构**，支持多租户、多客户定制化部署。

**核心目标**:
- 🔐 **安全可控**: License授权 + 账号数控制 + 四层权限体系
- 🧩 **灵活扩展**: 模块化设计 + 插件化架构 + 客户定制化
- 🚀 **高性能**: Java 21虚拟线程 + 分库分表 + 多级缓存
- 📊 **可观测**: 操作日志 + 链路追踪 + 实时监控

**数据权限范围定义**:

| 编码 | 名称 | SQL条件示例 |
|------|------|------------|
| 1 | 全部数据 | 无条件 |
| 2 | 本组织及下级 | `org_id IN (SELECT id FROM sys_org WHERE ancestors LIKE CONCAT(#{orgId},'%'))` |
| 3 | 仅本组织 | `org_id = #{orgId}` |
| 4 | 本部门及下级 | `dept_id IN (SELECT id FROM sys_dept WHERE ancestors LIKE CONCAT(#{deptId},'%'))` |
| 5 | 仅本部门 | `dept_id = #{deptId}` |
| 6 | 仅本人 | `create_by = #{userId}` |
| 7 | 自定义 | 根据规则表达式动态生成 |

## 3. 非功能需求

### 3.1 性能指标

| 指标 | 目标值 | 测试条件 |
|------|--------|---------|
| 接口响应时间(P95) | < 200ms | 500并发用户 |
| 接口响应时间(P99) | < 500ms | 500并发用户 |
| 吞吐量(TPS) | > 1000 | 简单查询接口 |
| 数据库查询 | < 50ms | 单表10万数据 |
| 系统可用性 | 99.9% | 年度统计 |
| 启动时间 | < 30s | 正常启动 |

### 3.2 安全要求

| 类别 | 要求 |
|------|------|
| 传输安全 | HTTPS强制, TLS 1.2+ |
| 密码存储 | BCrypt (strength=12) |
| 敏感数据 | AES-256加密存储 |
| 会话安全 | JWT签名 + Redis存储 |
| 防护措施 | SQL注入/XSS/CSRF防护 |
| 日志脱敏 | 敏感字段自动脱敏 |
| 审计追踪 | 关键操作全链路记录 |

### 3.3 可扩展性要求

| 要求 | 说明 |
|------|------|
| 模块热插拔 | 模块可独立启用/禁用 |
| 客户定制 | 支持覆盖标准实现 |
| 水平扩展 | 无状态设计,支持多实例 |
| 数据分片 | 支持表级水平分片 |
