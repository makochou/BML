# API 平台现状说明

## 1. 当前实现重点

当前 API 平台的核心目标是先把“开放接口安全链路”做扎实，而不是先堆页面功能。

当前已经落地的能力：

- API 账号创建、编辑、删除
- Secret 一次性返回
- Secret 加密存储
- OpenAPI 签名校验
- `nonce` 防重放
- 账号到接口的授权校验

## 2. 当前前端状态

- 应用管理页保留并可用
- API 管理页当前为占位页
- 在线调试页当前为占位页

## 3. 设计原则

- 先保证安全正确，再考虑门户体验
- Secret 只在创建或重置时展示一次
- 列表和详情接口默认不回传 Secret
- 账号有效期、状态、授权范围必须全部后端强校验

## 4. 后续演进方向

- 完整 API 目录页
- 在线调试页与真实开放接口联动
- 接口文档与授权关系可视化
