# API账号管理与授权指南

## 1. 功能目标

`API账号管理` 页面用于在 BML 中台统一维护项目接口的接入账号，并完成以下闭环：

- 新建、编辑、启停、删除 API 账号
- 重置 `AccessKey / SecretKey`
- 为账号授权可访问的项目受管接口
- 维护接入治理字段，避免关键信息散落在备注中
- 维护安全治理字段，包括签名版本、按环境独立 IP 白名单、回调地址
- 查看回调日志、手动发送测试回调、失败后立即重试

系统当前统一使用以下主表：

- `sys_api_account`
  - API 账号主表
- `sys_api_registry`
  - 项目接口目录主表
- `sys_api_permission`
  - 账号与项目接口授权关系表
- `sys_api_callback_log`
  - API 回调日志与重试记录表

## 2. 账号治理字段

### 2.1 接入方治理字段

- `owner_name`
  - 接入方负责人
- `owner_contact`
  - 负责人联系方式
- `system_name`
  - 业务系统名称
- `system_code`
  - 业务系统编码
- `access_environment`
  - 接入环境，固定为 `test / staging / production`

### 2.2 调用客户端字段

- `client_types`
  - 调用该账号的客户端类型
  - 当前支持：`web / h5 / app / mini_program / server / third_party / other`

### 2.3 安全治理字段

- `sign_version`
  - 签名算法版本
  - 当前正式支持 `v1`
- `callback_url`
  - 业务回调地址
- `test_ip_whitelist`
  - 测试环境独立白名单
- `staging_ip_whitelist`
  - 预发环境独立白名单
- `production_ip_whitelist`
  - 生产环境独立白名单
- `ip_whitelist`
  - 当前生效白名单的兼容字段
  - 系统会根据 `access_environment` 自动回填对应环境白名单

## 3. 为什么要按环境独立维护白名单

如果测试、预发、生产共用一份白名单，会带来明显风险：

- 测试环境机器可能误入生产白名单
- 预发与生产出口 IP 不同，联调时容易频繁修改配置
- 无法快速判断某个账号在当前环境真正生效的是哪一组白名单

因此本期将白名单拆分成三组独立字段，并遵循以下规则：

- 表单中按环境分别维护
- 运行时根据 `access_environment` 选择实际生效的白名单
- 旧字段 `ip_whitelist` 保留用于兼容旧链路，但写入时由系统统一计算

## 4. 回调日志与重试机制

### 4.1 设计目标

回调地址不再只是登记字段，系统已经具备统一回调日志和失败重试能力，便于后续业务模块复用。

### 4.2 日志表说明

`sys_api_callback_log` 记录以下关键信息：

- 账号快照：账号 ID、账号名称、系统名称、系统编码
- 回调快照：回调地址、请求头、请求体、响应状态码、响应体
- 过程状态：待执行、重试中、成功、失败
- 重试信息：已重试次数、最大重试次数、下次重试时间
- 时间信息：创建时间、最近回调时间、成功时间
- 错误信息：最近一次错误消息

### 4.3 当前支持的能力

- 管理端手动发送测试回调
- 查询指定账号的回调日志分页
- 管理端对单条失败/重试中日志执行“立即重试”
- 定时扫描到期的重试任务并自动重试

### 4.4 重试策略

当前默认策略如下，均可通过配置调整：

- 是否开启自动重试：`bml.openapi.callback.retry-enabled`
- 最大重试次数：`bml.openapi.callback.max-retry-count`
- 重试间隔秒数：`bml.openapi.callback.retry-interval-seconds`
- 扫描固定延迟：`bml.openapi.callback.processor-fixed-delay-ms`
- 单批次重试数量：`bml.openapi.callback.batch-size`

默认行为：

- 2xx 视为回调成功
- 非 2xx 或调用异常视为失败
- 若允许继续重试，则状态切换为“重试中”
- 达到最大次数后，状态切换为“失败”

## 7. 字段校验规则

### 7.1 业务系统编码 `system_code`

- 长度 `2-64`
- 仅允许字母、数字、下划线、中划线
- 保存时统一转为大写

### 7.2 接入环境 `access_environment`

固定值：

- `test`
- `staging`
- `production`

### 7.3 三环境白名单

支持：

- 单个 IP，例如 `203.0.113.10`
- CIDR，例如 `10.0.0.0/24`

输入规则：

- 支持换行、逗号、中文逗号、分号混合输入
- 保存时自动去空值、去重、标准化

### 7.4 签名算法版本 `sign_version`

当前正式支持：

- `v1`

### 7.5 回调地址 `callback_url`

- 可为空
- 如填写，必须是合法 `http://` 或 `https://` 地址

## 8. 全项目接口安全校验链路

受纳管项目接口统一使用 API 账号签名鉴权或后台 JWT 鉴权：

- 后台用户访问：继续走 JWT + 菜单/按钮权限
- 外部系统访问：走 `X-Bml-App-Key + 签名` 鉴权

默认纳管范围：

- 项目内 `com.bml.*` 控制器下的业务接口
- 只要不是登录、刷新令牌、健康检查、Swagger、错误页等基础设施路径
- 新增控制器方法后，应用启动时会自动同步进入授权目录，也可手动点击“同步项目接口”

必须携带的请求头：

- `X-Bml-App-Key`
- `X-Bml-Sign-Version`
- `X-Bml-Timestamp`
- `X-Bml-Nonce`
- `X-Bml-Sign`

系统在 API 账号认证过滤器与 `OpenApiInterceptor` 中统一校验：

1. 账号是否存在
2. 账号是否启用
3. 账号是否过期
4. 接口是否已授权
5. 签名版本是否匹配
6. 来源 IP 是否命中当前环境白名单
7. 时间戳是否超时
8. Nonce 是否重复
9. 签名是否正确

新增错误码：

- `2108`
  - 签名算法版本不匹配
- `2109`
  - 当前来源 IP 不在白名单内

## 9. 操作教程

### 9.1 新建 API 账号

1. 打开 `API账号管理`
2. 点击“新建账号”
3. 填写基础字段、治理字段、安全字段
4. 保存后记录一次性返回的 `AccessKey / SecretKey`
5. 进入“接口授权”工作台勾选项目接口

### 9.2 配置环境白名单

1. 先确认客户测试、预发、生产的真实出口 IP
2. 分别填写三套白名单
3. 设置账号当前接入环境
4. 保存后以该环境的白名单作为当前生效白名单

### 9.3 配置回调地址

1. 登记客户已确认可接收回调的地址
2. 保存后可在“回调日志”中发送测试回调
3. 联调失败时，可进入日志抽屉查看响应并手动重试

### 9.4 配置接口授权

1. 点击表格操作列中的“授权”
2. 在授权工作台中勾选接口
3. 点击“保存授权”

## 10. 常见问题排查

### 10.1 返回 401

优先检查：

- 请求头是否完整
- 时间戳是否超时
- Nonce 是否重复
- 签名版本是否正确
- `SecretKey` 是否已被重置
- 账号是否被停用或过期

### 10.2 返回 403

通常表示：

- 账号已认证通过，但接口未授权
- 或来源 IP 不在当前环境白名单中

### 10.3 新接口没有出现在授权树中

检查：

- 是否属于 `com.bml.*` 包下的控制器
- 是否命中了登录、文档、健康检查等基础设施排除路径
- 是否能解析出唯一 HTTP Method
- 是否执行过“同步项目接口”

### 10.4 回调一直失败

检查：

- 回调地址是否可达
- 业务系统是否已开放对应路径
- 响应是否返回 2xx
- 当前日志是否仍允许继续重试

注意事项：

1. 已落库的 Flyway 迁移文件不要直接改写
2. 如需修正逻辑，必须新增更高版本迁移

## 12. 手工验收清单

- 能按账号名称、状态、类型、客户端、业务系统、环境筛选
- 新建账号时可维护三环境白名单、签名版本、回调地址
- 列表可看到账号 ID、系统归属、负责人、环境、签名版本、白名单摘要、回调地址
- 凭证弹窗能显示三环境白名单与当前生效白名单
- 带错误签名版本的请求会被拒绝
- 来源 IP 不在当前环境白名单中的请求会被拒绝
- 打开授权工作台后可正常筛选、勾选并保存接口
- 回调日志抽屉可查看日志分页、统计摘要
- 可以发送测试回调
- 失败日志可以“立即重试”
- 自动重试任务启动后可处理到期重试日志
