## 测试场景

### 场景 1: 长授权树滚动条位置检查

**操作步骤**:
1. 在浏览器中打开 BML 企业管理系统
2. 导航到 API 账号管理页面
3. 选择一个有 50+ 接口的 API 账号
4. 点击"授权"按钮打开授权配置抽屉
5. 打开浏览器开发者工具（F12）
6. 使用元素选择器检查以下元素：
   - `.arco-drawer-body`
   - `.authorization-workbench`
   - `.governance-workbench__main`
   - `.authorization-panel--tree`
   - `.authorization-tree-shell`
7. 观察哪个元素上出现了滚动条

**预期观察（Bug 表现）**:
- ❌ 滚动条出现在 `.arco-drawer-body` 或其他外层容器上
- ❌ 滚动条**不是**仅出现在 `.authorization-tree-shell` 上
- ❌ 整个抽屉内容可以滚动

**期望行为（修复后）**:
- ✅ 滚动条应该**仅**出现在 `.authorization-tree-shell` 上
- ✅ 抽屉整体应保持固定高度，不出现滚动条

### 场景 2: 工具栏可见性测试

**操作步骤**:

**预期观察（Bug 表现）**:

**期望行为（修复后）**:
- ✅ 筛选工具栏应保持固定在顶部，始终可见

## 开发者工具检查

### 检查 Flex 布局链

**操作步骤**:
1. 使用开发者工具的元素选择器
2. 依次检查以下元素的计算样式（Computed Styles）：
   - `.arco-drawer-body`
   - `.authorization-workbench`
   - `.governance-workbench__main`
   - `.authorization-panel--tree`
   - `.authorization-tree-shell`

**需要检查的 CSS 属性**:
- `display`
- `flex-direction`
- `flex`
- `min-height`
- `height`
- `overflow-y`

### 预期发现（根本原因）

根据设计文档的假设，应该观察到以下问题：

1. **缺少高度限制**:
   - `.authorization-panel--tree` 可能缺少 `min-height: 0`
   - 导致其高度随内容增长而无限扩展

2. **滚动容器层级错误**:
   - `.authorization-tree-shell` 虽然设置了 `overflow-y: auto`
   - 但由于父容器没有高度限制，滚动条实际出现在更外层

3. **Flex 布局链断裂**:
   - `.authorization-panel--tree` 可能没有正确参与 flex 收缩
   - 导致其高度不受约束

4. **现有样式不完整**:
   - 代码中已有一些尝试修复的样式
   - 但这些样式与实际需要的 flex 布局约束不完全匹配
