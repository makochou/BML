# Preservation Property Test - Property 2

**Property 2: Preservation** - 所有现有功能不变

**测试目标**: 在未修复的代码上观察和记录所有非 bug 相关的功能行为，建立保留基准

**测试方法**: 手动功能测试 + 视觉回归测试

**重要说明**: 
- ⚠️ 此测试应在**未修复的代码**上运行
- ⚠️ 测试**预期通过**（确认基线行为需要保留）
- ✅ 记录所有观察到的功能行为作为保留基准
- ✅ 修复后重新运行此测试，确认所有功能行为保持不变

**Validates: Requirements 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7**

### 场景 1: 勾选和取消勾选接口功能

**测试目标**: 验证用户在授权树中勾选或取消勾选接口时，系统正常更新选中状态和统计数据

**操作步骤**:
1. 在浏览器中打开 BML 企业管理系统
2. 导航到 API 账号管理页面
3. 选择一个 API 账号，点击"授权"按钮打开授权配置抽屉
4. 在授权树中找到一个未勾选的接口
5. 点击该接口的复选框进行勾选
6. 观察以下内容：
   - 复选框状态变化（未勾选 → 勾选）
   - 侧边栏"授权数据概览"中的"已选接口"数量是否增加
   - 父节点（控制器、模块）的复选框状态变化（如适用）
7. 再次点击该接口的复选框取消勾选
8. 观察以下内容：
   - 复选框状态变化（勾选 → 未勾选）
   - 侧边栏"授权数据概览"中的"已选接口"数量是否减少
   - 父节点（控制器、模块）的复选框状态变化（如适用）

**预期行为（基线）**:
- ✅ 勾选接口后，复选框立即显示为勾选状态
- ✅ 侧边栏"已选接口"数量实时更新（+1）
- ✅ 如果控制器下所有接口都被勾选，控制器节点也显示为勾选状态
- ✅ 如果模块下所有接口都被勾选，模块节点也显示为勾选状态
- ✅ 取消勾选接口后，复选框立即显示为未勾选状态
- ✅ 侧边栏"已选接口"数量实时更新（-1）
- ✅ 父节点的复选框状态根据子节点状态正确更新
