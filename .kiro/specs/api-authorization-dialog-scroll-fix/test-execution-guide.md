# 测试执行指南

## Task 1: Bug Condition 探索性测试

### 测试文档位置
`.kiro/specs/api-authorization-dialog-scroll-fix/bug-exploration-test.md`

### 测试类型
**手动浏览器测试** - 此测试需要在真实浏览器环境中执行，因为它涉及：
- 视觉滚动行为观察
- CSS 布局和计算样式检查
- 用户交互体验验证

### 执行步骤

#### 1. 启动开发服务器

```bash
# 在 bml-frontend 目录下
cd bml-frontend
npm run dev
# 或
yarn dev
```

#### 2. 在浏览器中打开应用

访问开发服务器地址（通常是 `http://localhost:3000` 或类似地址）

#### 3. 导航到测试页面

1. 登录 BML 企业管理系统
2. 导航到 **API 账号管理** 页面
3. 找到一个有 **50+ 接口** 的 API 账号
4. 点击该账号的 **"授权"** 按钮

#### 4. 执行测试场景

按照 `bug-exploration-test.md` 中的 4 个测试场景依次执行：

**场景 1**: 检查滚动条出现的位置
- 打开开发者工具（F12）
- 使用元素选择器检查各个容器
- 记录滚动条实际出现的位置

**场景 2**: 测试工具栏可见性
- 向下滚动授权树
- 观察顶部筛选工具栏是否被滚动到视口外

**场景 3**: 测试侧边栏可见性
- 向下滚动授权树
- 观察左侧边栏是否被滚动到视口外

**场景 4**: 测试短授权树
- 切换到只有 3 个接口的账号
- 观察是否出现不必要的滚动条

#### 5. 记录测试结果

在 `bug-exploration-test.md` 的 **测试结果记录** 部分填写：
- 勾选实际观察到的行为
- 填写 CSS 属性检查结果
- 确认或反驳根本原因假设

### 预期结果

⚠️ **重要**: 此测试在未修复的代码上**预期失败**

预期观察到的 Bug 表现：
- ❌ 滚动条出现在抽屉整体而非 `.authorization-tree-shell`
- ❌ 工具栏和侧边栏会被滚动到视口外
- ❌ 整个抽屉内容可以滚动

这些失败确认了 bug 的存在，是**正常且预期的结果**。

### 测试完成标准

- [x] 测试文档已创建
- [ ] 开发服务器已启动
- [ ] 测试已在浏览器中执行
- [ ] 所有 4 个场景已测试
- [ ] 测试结果已记录在 `bug-exploration-test.md`
- [ ] Bug 存在已确认

### 下一步

完成测试后：
1. **不要尝试修复代码** - 这是探索性测试
2. 将测试结果报告给协调器
3. 等待 Task 2（保留性测试）
4. 等待 Task 3（实施修复）
5. 修复后将重新运行此测试验证效果
