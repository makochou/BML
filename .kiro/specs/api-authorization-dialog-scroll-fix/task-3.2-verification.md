## 测试场景验证

### 场景 1: 长授权树滚动条位置检查 ✅

**操作步骤**:
1. 打开授权配置抽屉（包含 50+ 接口）
2. 打开浏览器开发者工具（F12）
3. 使用元素选择器检查以下元素：
   - `.arco-drawer-body`
   - `.authorization-workbench`
   - `.governance-workbench__main`
   - `.authorization-panel--tree`
   - `.authorization-tree-shell`
4. 观察哪个元素上出现了滚动条

**期望结果（修复后）**:
- ✅ 滚动条应该**仅**出现在 `.authorization-tree-shell` 上
- ✅ 抽屉整体应保持固定高度，不出现滚动条
- ✅ 外层容器（`.arco-drawer-body`、`.authorization-workbench` 等）不应出现滚动条

**验证要点**:
- 使用开发者工具的元素选择器，hover 到 `.authorization-tree-shell` 元素
- 确认该元素的 `overflow-y` 计算样式为 `auto` 或 `scroll`
- 确认该元素内部有滚动条（当内容超出时）
- 确认外层容器没有滚动条

---

### 场景 2: 工具栏可见性测试 ✅

1. 在授权抽屉中向下滚动查看授权树底部的接口

**期望结果（修复后）**:
- ✅ 筛选工具栏应保持固定在顶部，始终可见
- ✅ 只有授权树内容滚动，工具栏不滚动
- ✅ 无论滚动到授权树的任何位置，工具栏都应保持在视口内

**验证要点**:
- 滚动到授权树底部
- 确认顶部的搜索框、模块下拉、方法下拉仍然可见
- 确认可以随时使用筛选功能，无需滚动回顶部

## CSS 属性验证

### 使用开发者工具检查计算样式

**`.authorization-panel--tree` 应该具有**:
```css
display: flex
flex-direction: column
min-height: 0
flex: 1 1 0%
```

**`.authorization-toolbar` 和 `.section-heading` 应该具有**:
```css
flex-shrink: 0
```

**`.authorization-tree-shell` 应该具有**:
```css
flex: 1 1 0%
min-height: 0
overflow-y: auto
```
