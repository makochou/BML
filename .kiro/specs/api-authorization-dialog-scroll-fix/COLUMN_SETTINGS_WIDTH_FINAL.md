# 列设置弹窗宽度最终调整

## 问题描述

用户通过新的红线标注明确指出，列设置弹窗的宽度需要调整到更窄的位置，大约在"负责人"列的右侧边缘。

## 解决方案

### 1. 弹窗宽度大幅缩减

**调整前**：
- `width: 350px` - 宽度过宽，超出红线标注

**调整后**：
- `width: 280px` - 精确对齐到红线位置

**缩减幅度**：
- 从 350px 减少到 280px
- 缩减 70px，减少 20%

### 2. 内边距调整

**调整前**：
- `padding: 12px 14px`

**调整后**：
- `padding: 10px 12px`

**原因**：
- 适应更窄的宽度
- 保持内容与边框的合理间距

### 3. 列表项间距优化

**调整前**：
- `gap: 6px` - 主容器间距
- `padding: 6px 10px` - 列表项内边距

**调整后**：
- `gap: 4px` - 主容器间距，减少 33%
- `padding: 6px 8px` - 列表项内边距，减少 20%

**原因**：
- 适应更窄的宽度
- 保持紧凑但不拥挤的视觉效果

### 4. 标签区域间距优化

**调整前**：
- `gap: 6px` - 拖拽手柄和文字之间的间距

**调整后**：
- `gap: 4px` - 减少 33%

**原因**：
- 适应更窄的宽度
- 保持元素之间的合理间距

### 5. 按钮区域间距优化

**调整前**：
- `gap: 5px` - 按钮之间的间距

**调整后**：
- `gap: 3px` - 减少 40%

**原因**：
- 适应更窄的宽度
- 保持按钮之间的最小间距

### 6. 字体大小调整

**调整前**：
- 字段名称：`font-size: 13px`
- 固定标签：`font-size: 10px`

**调整后**：
- 字段名称：`font-size: 12px` - 减少 1px
- 固定标签：`font-size: 9px` - 减少 1px

**原因**：
- 适应更窄的宽度
- 保持可读性

### 7. 按钮尺寸调整

**调整前**：
- 按钮尺寸：`22px × 22px`
- 拖拽手柄：`18px × 18px`

**调整后**：
- 按钮尺寸：`20px × 20px` - 减少 2px
- 拖拽手柄：`16px × 16px` - 减少 2px

**原因**：
- 适应更窄的宽度
- 保持可点击性

### 8. 按钮圆角调整

**调整前**：
- `border-radius: 5px`

**调整后**：
- `border-radius: 4px`

**原因**：
- 与更小的按钮尺寸匹配
- 保持视觉协调

## 宽度计算详解

### 内容区域（280px）
```
拖拽手柄：16px
字段名称：约 80px（最长字段，会被截断）
固定标签：约 30px
上下按钮：20px × 2 = 40px
固定按钮：20px
开关按钮：约 36px
间距：4px × 5 = 20px
小计：242px

边距：
左右内边距：12px × 2 = 24px
边框：1px × 2 = 2px
小计：26px

总宽度：
内容 + 边距：242px + 26px = 268px
预留空间：12px（用于微调）
最终宽度：280px
```
