# 告警去重功能 - 快速开始

## 📋 概述

告警去重功能是针对许可证更新时重复告警问题的优化方案。通过智能去重机制，确保每种类型的许可证变更只显示最新的一条告警，避免重复提醒用户。

## 🎯 核心特性

- ✅ **智能去重**：自动检测相同类型和标题的未读告警，更新而非创建新记录
- ✅ **保持未读**：更新告警时保持未读状态，确保用户不会错过重要信息
- ✅ **历史保留**：已读告警不影响新告警的创建，保留完整的历史记录
- ✅ **性能优化**：通过数据库索引优化查询性能，性能影响可忽略
- ✅ **向后兼容**：不影响现有功能，平滑升级

## 🚀 快速开始

### 1. 数据库迁移

系统启动时会自动执行数据库迁移脚本，无需手动干预：

```sql
-- V1.9.6__optimize_alert_deduplication.sql
CREATE INDEX idx_alert_dedup ON sys_alert (alert_type, alert_title, read_status, deleted);
```

### 2. 代码集成

告警保存时自动使用智能去重方法：

```java
// 原有方式（已废弃）
sysAlertService.save(alert);

// 新方式（自动去重）
sysAlertService.saveOrUpdateAlert(alert);
```

### 3. 前端集成

前端无需修改，告警展示逻辑保持不变。

## 🔧 配置说明

### 去重规则

- **去重维度**：`alertType` + `alertTitle` + `readStatus = 0`
- **去重范围**：仅对未读告警进行去重
- **更新策略**：更新内容和级别，保持未读状态

### 不去重的场景

1. **已读告警**：如果相同类型和标题的告警已被用户标记为已读，则创建新记录
2. **不同标题**：即使类型相同，标题不同的告警会分别创建
3. **不同类型**：即使标题相同，类型不同的告警会分别创建

## 🛠️ 故障排查

### 问题 1：告警没有去重

**可能原因**：

1. 告警类型或标题不完全相同
2. 旧告警已被标记为已读
3. 数据库索引未创建成功

**解决方案**：

1. 检查告警类型和标题是否完全一致
2. 确认旧告警的读取状态
3. 手动执行数据库迁移脚本

### 问题 2：性能下降

**可能原因**：

1. 数据库索引未创建
2. 数据库连接池配置不当
3. 告警数据量过大

**解决方案**：

1. 检查索引是否存在：`SHOW INDEX FROM sys_alert;`
2. 优化数据库连接池配置
3. 定期清理历史告警数据

### 问题 3：告警丢失

**可能原因**：

1. 告警被误删除
2. 数据库逻辑删除标记错误

**解决方案**：

1. 检查数据库中的 `deleted` 字段
2. 恢复误删除的告警：`UPDATE sys_alert SET deleted = 0 WHERE id = ?;`
