# 告警批量管理功能设计

## 📋 需求分析

### 用户痛点

从截图可以看到：
1. 告警中心显示了 **14 条相同类型的告警**（API 账号上限升级）
2. 用户需要逐条点击"标记已读"或"删除"，操作繁琐
3. 用户希望能够**一键清理**相同类型的告警

### 解决方案

提供以下批量操作功能：
1. **按类型批量标记已读**：一键标记所有相同类型的告警为已读
2. **按类型批量删除**：一键删除所有相同类型的告警
3. **按标题批量标记已读**：一键标记所有相同标题的告警为已读
4. **按标题批量删除**：一键删除所有相同标题的告警

 * 按告警类型批量删除

/**
 * 按告警标题批量标记为已读

 * @param alertTitle 告警标题
 * @return 更新的记录数

/**
 * 按告警标题批量删除

#### 1.2 Service 实现

```java
@Override
public int markAsReadByType(String alertType) {
    LambdaUpdateWrapper<SysAlert> updateWrapper = new LambdaUpdateWrapper<>();
    updateWrapper.eq(SysAlert::getAlertType, alertType)
            .eq(SysAlert::getReadStatus, 0)
            .set(SysAlert::getReadStatus, 1);
    int count = this.baseMapper.update(null, updateWrapper);
    log.info("[Alert] 按类型批量标记已读：类型={}, 数量={}", alertType, count);
    return count;
}

@Override
public int deleteByType(String alertType) {
    LambdaQueryWrapper<SysAlert> queryWrapper = new LambdaQueryWrapper<>();
    queryWrapper.eq(SysAlert::getAlertType, alertType);
    int count = this.baseMapper.delete(queryWrapper);
    log.info("[Alert] 按类型批量删除：类型={}, 数量={}", alertType, count);
    return count;
}

@Override
public int markAsReadByTitle(String alertTitle) {
    LambdaUpdateWrapper<SysAlert> updateWrapper = new LambdaUpdateWrapper<>();
    updateWrapper.eq(SysAlert::getAlertTitle, alertTitle)
            .eq(SysAlert::getReadStatus, 0)
            .set(SysAlert::getReadStatus, 1);
    int count = this.baseMapper.update(null, updateWrapper);
    log.info("[Alert] 按标题批量标记已读：标题={}, 数量={}", alertTitle, count);
    return count;
}

@Override
public int deleteByTitle(String alertTitle) {
    LambdaQueryWrapper<SysAlert> queryWrapper = new LambdaQueryWrapper<>();
    queryWrapper.eq(SysAlert::getAlertTitle, alertTitle);
    int count = this.baseMapper.delete(queryWrapper);
    log.info("[Alert] 按标题批量删除：标题={}, 数量={}", alertTitle, count);
    return count;
}
```

### 2. 前端实现

#### 2.1 Vue 3 组件

```vue
<template>
  <div class="alert-item">
    <!-- 告警卡片 -->
    <div class="alert-header">
      <span class="alert-title">{{ alert.alertTitle }}</span>
      <el-dropdown @command="handleBatchCommand">
        <el-icon><MoreFilled /></el-icon>
        <template #dropdown>
          <el-dropdown-menu>
            <el-dropdown-item command="readByType">
              标记所有相同类型为已读
            </el-dropdown-item>
            <el-dropdown-item command="deleteByType">
              删除所有相同类型
            </el-dropdown-item>
            <el-dropdown-item divided command="readByTitle">
              标记所有相同标题为已读
            </el-dropdown-item>
            <el-dropdown-item command="deleteByTitle">
              删除所有相同标题
            </el-dropdown-item>
          </el-dropdown-menu>
        </template>
      </el-dropdown>
    </div>
    <!-- ... 其他内容 ... -->
  </div>
</template>

<script setup>
import { ElMessageBox, ElMessage } from 'element-plus';
import { MoreFilled } from '@element-plus/icons-vue';

const props = defineProps({
  alert: Object
});

const emit = defineEmits(['refresh']);

const handleBatchCommand = async (command) => {
  let confirmMessage = '';
  let apiUrl = '';
  let params = {};

  switch (command) {
    case 'readByType':
      confirmMessage = `确定要将所有"${props.alert.alertType}"类型的告警标记为已读吗？`;
      apiUrl = '/system/alert/read-by-type';
      params = { alertType: props.alert.alertType };
      break;
    case 'deleteByType':
      confirmMessage = `确定要删除所有"${props.alert.alertType}"类型的告警吗？此操作不可恢复！`;
      apiUrl = '/system/alert/delete-by-type';
      params = { alertType: props.alert.alertType };
      break;
    case 'readByTitle':
      confirmMessage = `确定要将所有"${props.alert.alertTitle}"的告警标记为已读吗？`;
      apiUrl = '/system/alert/read-by-title';
      params = { alertTitle: props.alert.alertTitle };
      break;
    case 'deleteByTitle':
      confirmMessage = `确定要删除所有"${props.alert.alertTitle}"的告警吗？此操作不可恢复！`;
      apiUrl = '/system/alert/delete-by-title';
      params = { alertTitle: props.alert.alertTitle };
      break;
  }

  try {
    await ElMessageBox.confirm(confirmMessage, '批量操作确认', {
      confirmButtonText: '确定',
      cancelButtonText: '取消',
      type: 'warning'
    });

    const response = await fetch(`${apiUrl}?${new URLSearchParams(params)}`, {
      method: command.includes('delete') ? 'DELETE' : 'PUT'
    });
    const result = await response.json();

    if (result.code === 200) {
      ElMessage.success(`成功处理 ${result.data} 条告警`);
      emit('refresh');
    } else {
      ElMessage.error(result.message || '操作失败');
    }
  } catch (error) {
    if (error !== 'cancel') {
      ElMessage.error('操作失败');
    }
  }
};
</script>
```

## 🎯 使用场景

### 场景 1：清理重复的许可证变更告警

**问题**：用户多次调整许可证配额，产生了大量相同类型的告警

**解决**：
1. 点击任意一条"API 账号上限升级"告警的下拉菜单
2. 选择"删除所有相同标题"
3. 确认操作
4. 所有相同标题的告警被一键清理

### 场景 2：批量标记已读

**问题**：用户查看了告警内容，希望批量标记为已读

**解决**：
1. 点击任意一条告警的下拉菜单
2. 选择"标记所有相同类型为已读"
3. 确认操作
4. 所有相同类型的告警被标记为已读
