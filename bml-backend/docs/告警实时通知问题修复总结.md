# 告警实时通知问题修复总结

## 📋 问题描述

### 用户反馈

> "现在咋没告警提示了呢，我重新授权API账号数增加到20了，但是系统没提示"

### 问题现象

1. 用户更新许可证，API 账号数从 10 个增加到 20 个
2. 告警中心显示的告警内容已更新：「API 账号上限升级：许可证更新：最大 API 账号数从 5 变更为 10，10 变更为 20。」
3. 但是前端没有弹出实时通知（Toast 提示）
4. 用户需要手动刷新告警中心才能看到更新后的内容

### 问题截图分析

从用户提供的截图可以看到：
- 告警中心显示了「API 账号上限升级」告警
- 告警内容确实已更新
- 但用户没有收到实时通知

---

## 🔍 问题分析

### 根本原因

告警去重机制会**更新**现有告警而不是创建新告警，但原有的增量轮询接口基于 `lastId`（告警ID）来判断是否有新告警。当告警被更新时，ID 不会变化，导致前端无法感知到告警更新。

### 详细分析

#### 场景重现

1. **第一次更新许可证**：API 账号从 5 个升级到 10 个
   - 系统创建告警（ID=1）：「API 账号上限升级：从 5 变更为 10」
   - 前端轮询获取到这条告警，记录 `lastId = 1`
   - 前端显示实时通知 ✅

2. **第二次更新许可证**：API 账号从 10 个升级到 20 个
   - 系统**更新**告警（ID=1）：「API 账号上限升级：从 10 变更为 20」
   - 告警的 `update_time` 字段更新为当前时间
   - 但告警的 `id` 仍然是 1

3. **前端轮询**：
   - 前端携带 `lastId = 1` 请求 `/system/alert/latest?lastId=1`
   - 后端执行查询：`SELECT * FROM sys_alert WHERE id > 1 AND deleted = 0`
   - **结果**：没有返回任何数据（因为告警 ID 仍然是 1，不大于 1）
   - 前端没有收到新告警，无法显示实时通知 ❌

#### 问题根源

- **告警去重机制**：为了避免重复告警，系统会更新现有告警而不是创建新告警
- **增量轮询机制**：基于 ID 的增量查询无法感知到告警更新（ID 不变）
- **矛盾点**：去重机制和轮询机制的设计冲突

---

## ✅ 解决方案

### 核心思路

将增量轮询从**基于 ID** 改为**基于时间戳**：

- **原方案**：`GET /system/alert/latest?lastId=1`
  - 查询逻辑：`WHERE id > 1`
  - 问题：告警更新时 ID 不变，无法感知

- **新方案**：`GET /system/alert/latest?lastPollTime=1714449600000`
  - 查询逻辑：`WHERE update_time > '2026-04-30 10:00:00'`
  - 优势：告警更新时 `update_time` 会变化，能及时感知

### 实现细节

#### 1. 后端改动

##### 1.1 修改服务接口

```java
// ISysAlertService.java

// ❌ 原方案
List<SysAlert> getLatestAlerts(Long lastId);

// ✅ 新方案
List<SysAlert> getLatestAlerts(Long lastPollTime);
```

##### 1.2 修改服务实现

```java
// SysAlertServiceImpl.java

@Override
public List<SysAlert> getLatestAlerts(Long lastPollTime) {
    if (lastPollTime == null) {
        // 首次加载：返回最近的告警
        return this.list(new LambdaQueryWrapper<SysAlert>()
                .orderByDesc(SysAlert::getCreateTime)
                .last("LIMIT " + DEFAULT_LATEST_LIMIT));
    }
    
    // 将时间戳转换为 LocalDateTime
    LocalDateTime lastPollDateTime = LocalDateTime.ofInstant(
            Instant.ofEpochMilli(lastPollTime),
            ZoneId.systemDefault());
    
    // ✅ 增量查询：返回 update_time 大于 lastPollTime 的告警
    return this.list(new LambdaQueryWrapper<SysAlert>()
            .gt(SysAlert::getUpdateTime, lastPollDateTime)
            .orderByAsc(SysAlert::getUpdateTime));
}
```

##### 1.3 修改控制器接口

```java
// SysAlertController.java

@GetMapping("/latest")
public Result<List<SysAlertVO>> getLatestAlerts(
        @RequestParam(name = "lastPollTime", required = false) Long lastPollTime) {
    return Result.ok(AlertConverter.INSTANCE.toVOList(
            sysAlertService.getLatestAlerts(lastPollTime)));
}
```

##### 1.4 确保 update_time 正确设置

```java
// SysAlertServiceImpl.java

@Override
public SysAlert saveOrUpdateAlert(SysAlert alert) {
    SysAlert existingAlert = this.getOne(new LambdaQueryWrapper<SysAlert>()
            .eq(SysAlert::getAlertType, alert.getAlertType())
            .eq(SysAlert::getAlertTitle, alert.getAlertTitle())
            .eq(SysAlert::getReadStatus, 0)
            .last("LIMIT 1"));

    if (existingAlert != null) {
        // 更新现有告警（update_time 会自动更新）
        existingAlert.setAlertContent(alert.getAlertContent());
        existingAlert.setAlertLevel(alert.getAlertLevel());
        this.updateById(existingAlert);
        return existingAlert;
    } else {
        // 创建新告警，设置初始的 update_time
        LocalDateTime now = LocalDateTime.now();
        alert.setCreateTime(now);
        alert.setUpdateTime(now);
        this.save(alert);
        return alert;
    }
}
```

##### 1.5 添加数据库索引

```sql
-- V1.9.6__optimize_alert_deduplication.sql

-- 为 update_time 添加索引，优化增量查询性能
CREATE INDEX idx_update_time ON sys_alert (update_time, deleted);
```

#### 2. 前端改动

##### 2.1 修改轮询逻辑

```javascript
// ❌ 原方案：基于 lastId
let lastAlertId = null;

setInterval(async () => {
  const response = await fetch(`/system/alert/latest?lastId=${lastAlertId || ''}`);
  const result = await response.json();
  
  if (result.data && result.data.length > 0) {
    result.data.forEach(alert => showNotification(alert));
    lastAlertId = Math.max(...result.data.map(a => a.id));
  }
}, 30000);

// ✅ 新方案：基于 lastPollTime
let lastPollTime = null;

setInterval(async () => {
  const response = await fetch(`/system/alert/latest?lastPollTime=${lastPollTime || ''}`);
  const result = await response.json();
  
  if (result.data && result.data.length > 0) {
    result.data.forEach(alert => showNotification(alert));
    lastPollTime = Date.now(); // 更新为当前时间戳
  }
}, 30000);
```

## 📊 改动对比

### API 变更

| 项目 | 原方案 | 新方案 |
|------|--------|--------|
| 接口路径 | `/system/alert/latest` | `/system/alert/latest` |
| 参数名 | `lastId` | `lastPollTime` |
| 参数类型 | Long（告警ID） | Long（时间戳，毫秒） |
| 参数示例 | `1` | `1714449600000` |
| 查询条件 | `WHERE id > ?` | `WHERE update_time > ?` |
| 排序方式 | `ORDER BY id ASC` | `ORDER BY update_time ASC` |

### 数据库变更

| 项目 | 说明 |
|------|------|
| 新增索引 | `idx_update_time (update_time, deleted)` |
| 索引用途 | 优化增量轮询查询性能 |
| 预期效果 | 查询耗时 < 5ms |

### 代码变更

| 文件 | 改动类型 | 改动内容 |
|------|----------|----------|
| ISysAlertService.java | 修改方法签名 | `getLatestAlerts(Long lastId)` → `getLatestAlerts(Long lastPollTime)` |
| SysAlertServiceImpl.java | 修改查询逻辑 | 从基于 ID 查询改为基于时间戳查询 |
| SysAlertController.java | 修改接口参数 | `lastId` → `lastPollTime` |
| SysAlertServiceImpl.java | 修改保存逻辑 | 确保创建时设置 `update_time` |
| V1.9.6__optimize_alert_deduplication.sql | 新增索引 | 添加 `idx_update_time` 索引 |
