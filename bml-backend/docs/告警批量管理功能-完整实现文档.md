# 告警批量管理功能 - 完整实现文档

## 📋 功能概述

告警批量管理功能已完整实现，用户可以通过 API 快速批量处理相同类型或相同标题的告警，大幅提升告警管理效率。

### 核心功能

1. **按类型批量标记已读** - 一键标记所有相同类型的告警为已读
2. **按类型批量删除** - 一键删除所有相同类型的告警
3. **按标题批量标记已读** - 一键标记所有相同标题的告警为已读
4. **按标题批量删除** - 一键删除所有相同标题的告警

### 2. Service 实现（SysAlertServiceImpl.java）

#### 2.1 按类型批量标记已读

```java
@Override
public int markAsReadByType(String alertType) {
    LambdaUpdateWrapper<SysAlert> updateWrapper = new LambdaUpdateWrapper<>();
    updateWrapper.eq(SysAlert::getAlertType, alertType)
            .eq(SysAlert::getReadStatus, 0)
            .set(SysAlert::getReadStatus, 1);
    int count = this.baseMapper.update(null, updateWrapper);
    log.info("[Alert] 按类型批量标记已读：类型={}, 数量={}", alertType, count);
    return count;
}
```

**核心逻辑**：
- 使用 MyBatis-Plus 的 `LambdaUpdateWrapper` 构建更新条件
- 只更新未读状态（`readStatus = 0`）的告警
- 返回实际更新的记录数
- 记录详细的操作日志

#### 2.2 按类型批量删除

```java
@Override
public int deleteByType(String alertType) {
    LambdaQueryWrapper<SysAlert> queryWrapper = new LambdaQueryWrapper<>();
    queryWrapper.eq(SysAlert::getAlertType, alertType);
    int count = (int) this.count(queryWrapper);
    this.remove(queryWrapper);
    log.info("[Alert] 按类型批量删除：类型={}, 数量={}", alertType, count);
    return count;
}
```

**核心逻辑**：
- 先统计符合条件的记录数
- 使用 `remove()` 方法进行逻辑删除（由 `@TableLogic` 注解自动处理）
- 返回实际删除的记录数
- 记录详细的操作日志

### 3. Controller API（SysAlertController.java）

新增了 4 个 RESTful API 接口：

#### 3.1 按类型批量标记已读

```java
/**
 * 按告警类型批量标记为已读
 */
@Operation(summary = "按告警类型批量标记为已读")
@PreAuthorize("@ss.hasPermi('monitor:alert:edit')")
@PutMapping("/read-by-type")
public Result<Integer> markAsReadByType(
        @Parameter(description = "告警类型") @RequestParam("alertType") String alertType) {
    return Result.ok(sysAlertService.markAsReadByType(alertType));
}
```

**接口信息**：
- **请求方法**：PUT
- **请求路径**：`/system/alert/read-by-type`
- **请求参数**：`alertType`（告警类型）
- **权限要求**：`monitor:alert:edit`
- **返回数据**：更新的记录数

**请求示例**：
```bash
curl -X PUT "http://localhost:8080/system/alert/read-by-type?alertType=LICENSE_CHANGE" \
  -H "Authorization: Bearer {token}"
```

**响应示例**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": 14
}
```

### 1. 告警卡片下拉菜单设计

在每条告警卡片上添加下拉菜单按钮（三个点图标），提供批量操作选项：

```vue
<template>
  <div class="alert-card">
    <div class="alert-header">
      <span class="alert-icon">🔵</span>
      <span class="alert-title">{{ alert.alertTitle }}</span>
      
      <!-- 下拉菜单按钮 -->
      <el-dropdown @command="handleBatchCommand" trigger="click">
        <el-button type="text" :icon="MoreFilled" circle />
        <template #dropdown>
          <el-dropdown-menu>
            <el-dropdown-item command="readByType">
              <el-icon><Check /></el-icon>
              标记所有相同类型为已读
            </el-dropdown-item>
            <el-dropdown-item command="deleteByType">
              <el-icon><Delete /></el-icon>
              删除所有相同类型
            </el-dropdown-item>
            <el-dropdown-item divided />
            <el-dropdown-item command="readByTitle">
              <el-icon><Check /></el-icon>
              标记所有相同标题为已读
            </el-dropdown-item>
            <el-dropdown-item command="deleteByTitle">
              <el-icon><Delete /></el-icon>
              删除所有相同标题
            </el-dropdown-item>
          </el-dropdown-menu>
        </template>
      </el-dropdown>
    </div>
    
    <div class="alert-content">{{ alert.alertContent }}</div>
    <div class="alert-time">{{ formatTime(alert.createTime) }}</div>
  </div>
</template>

<script setup>
import { ElMessageBox, ElMessage } from 'element-plus';
import { MoreFilled, Check, Delete } from '@element-plus/icons-vue';
import { markAsReadByType, deleteByType, markAsReadByTitle, deleteByTitle } from '@/api/alert';

const props = defineProps({
  alert: {
    type: Object,
    required: true
  }
});

const emit = defineEmits(['refresh']);

/**
 * 处理批量操作命令
 */
const handleBatchCommand = async (command) => {
  let confirmMessage = '';
  let apiFunction = null;
  let params = {};

  switch (command) {
    case 'readByType':
      confirmMessage = `确定要将所有"${props.alert.alertType}"类型的告警标记为已读吗？`;
      apiFunction = markAsReadByType;
      params = { alertType: props.alert.alertType };
      break;
    case 'deleteByType':
      confirmMessage = `确定要删除所有"${props.alert.alertType}"类型的告警吗？此操作不可恢复！`;
      apiFunction = deleteByType;
      params = { alertType: props.alert.alertType };
      break;
    case 'readByTitle':
      confirmMessage = `确定要将所有"${props.alert.alertTitle}"的告警标记为已读吗？`;
      apiFunction = markAsReadByTitle;
      params = { alertTitle: props.alert.alertTitle };
      break;
    case 'deleteByTitle':
      confirmMessage = `确定要删除所有"${props.alert.alertTitle}"的告警吗？此操作不可恢复！`;
      apiFunction = deleteByTitle;
      params = { alertTitle: props.alert.alertTitle };
      break;
  }

  try {
    await ElMessageBox.confirm(confirmMessage, '批量操作确认', {
      confirmButtonText: '确定',
      cancelButtonText: '取消',
      type: 'warning'
    });

    const result = await apiFunction(params);
    
    if (result.code === 200) {
      ElMessage.success(`成功处理 ${result.data} 条告警`);
      emit('refresh');
    } else {
      ElMessage.error(result.message || '操作失败');
    }
  } catch (error) {
    if (error !== 'cancel') {
      console.error('批量操作失败:', error);
      ElMessage.error('操作失败，请稍后重试');
    }
  }
};

/**
 * 格式化时间
 */
const formatTime = (time) => {
  // 实现时间格式化逻辑
  return time;
};
</script>
```

## 📈 性能优化

### 1. 数据库索引

批量操作依赖以下索引（已在之前的迁移脚本中创建）：

```sql
-- 告警去重索引（支持按类型和标题查询）
CREATE INDEX idx_alert_dedup ON sys_alert(alert_type, alert_title, read_status, deleted);

-- 更新时间索引（支持增量轮询）
CREATE INDEX idx_update_time ON sys_alert(update_time, deleted);
```

### 2. 批量操作优化

- 使用 MyBatis-Plus 的批量更新/删除方法，一次 SQL 完成操作
- 避免循环调用单条记录的更新/删除方法
- 先统计再删除，减少数据库查询次数

### 3. 日志记录

所有批量操作都记录详细日志，便于问题排查：

