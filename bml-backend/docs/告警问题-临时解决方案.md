# 告警重复提示问题 - 临时解决方案

## 📋 当前问题

从截图可以看到，告警中心显示了 **14 条相同类型的告警**（API 账号上限升级），用户希望能够快速清理这些重复的告警。

## ✅ 临时解决方案（立即可用）

### 方案 1：使用"全部已读"功能

**操作步骤**：
1. 打开告警中心
2. 点击右上角的"全部已读"按钮
3. 所有告警会被标记为已读，不再显示红点提示

**优点**：
- ✅ 操作简单，一键完成
- ✅ 立即生效，无需等待

**缺点**：
- ❌ 告警仍然存在于列表中，只是标记为已读
- ❌ 如果有其他重要告警，也会被一起标记为已读

### 方案 2：逐条删除告警

**操作步骤**：
1. 打开告警中心
2. 找到要删除的告警
3. 点击"删除"按钮
4. 重复以上步骤，直到清理完所有重复告警

**优点**：
- ✅ 可以选择性删除
- ✅ 彻底清理，不留痕迹

**缺点**：
- ❌ 操作繁琐，需要逐条点击
- ❌ 如果有很多重复告警，耗时较长

### 方案 3：直接操作数据库（仅限管理员）

**操作步骤**：
1. 连接到数据库
2. 执行以下 SQL 语句：

```sql
-- 删除所有"API 账号上限升级"的告警
DELETE FROM sys_alert 
WHERE alert_title = 'API 账号上限升级' 
  AND deleted = 0;

-- 或者标记为已读
UPDATE sys_alert 
SET read_status = 1 
WHERE alert_title = 'API 账号上限升级' 
  AND deleted = 0;
```

**优点**：
- ✅ 快速高效，一次性处理所有重复告警
- ✅ 可以精确控制删除范围

**缺点**：
- ❌ 需要数据库访问权限
- ❌ 有一定风险，需要谨慎操作

### 立即操作（解决当前问题）

   - 使用"全部已读"功能，将所有告警标记为已读
